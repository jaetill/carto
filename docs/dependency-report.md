### `lambda/package.json` — Lambda (carto-lambda)

**Security audit:** 18 moderate vulnerabilities (221 prod deps audited).

#### Moderate security advisories (flag for update)

All 18 findings share a single root cause: **`@opentelemetry/core < 2.8.0`** pulled in transitively by `@sentry/aws-serverless ^9`.

> **GHSA-8988-4f7v-96qf** — OpenTelemetry Core: Unbounded memory allocation in W3C Baggage propagation (CVSS 5.3 / `CWE-770`).
> An unauthenticated remote caller can craft a request that causes unbounded memory growth in the baggage propagation layer, leading to a Lambda OOM condition.
> **Fix:** upgrade `@sentry/aws-serverless` to `^10.62.0` — this is a **major version bump** (9 → 10) and is the only available fix.

Affected transitive packages (all resolved by the same Sentry upgrade):

`@opentelemetry/core`, `@opentelemetry/instrumentation-amqplib`, `@opentelemetry/instrumentation-aws-sdk`,
`@opentelemetry/instrumentation-connect`, `@opentelemetry/instrumentation-express`,
